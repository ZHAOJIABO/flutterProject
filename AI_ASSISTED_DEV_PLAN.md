## 🔧 第三阶段：后端开发（可选）

**时间：第 17-24 周（80-160 小时）**

### 决策：是否需要自建后端？

#### ✅ 推荐使用 BaaS（无需自建后端）

**适用场景：**
- MVP 快速验证
- 用户量 < 10,000
- 预算有限
- 团队只有前端开发

**推荐方案：**

**1. Supabase（强烈推荐）**
- ✅ 开源，数据可导出
- ✅ PostgreSQL 数据库
- ✅ 实时订阅
- ✅ 文件存储
- ✅ 用户认证（支持社交登录）
- ✅ Row Level Security（数据安全）
- ✅ 免费额度：500MB 数据库、1GB 文件存储

**集成时间：1-2 周（10-20 小时）**

```yaml
# pubspec.yaml
dependencies:
  supabase_flutter: ^2.3.0
```

**2. Firebase（备选）**
- ✅ Google 官方支持
- ✅ 丰富的功能（推送、分析、Crashlytics）
- ✅ 免费额度较大
- ❌ 数据迁移困难
- ❌ NoSQL（Firestore）学习曲线

---

#### ⚠️ 需要自建后端的情况

**必须自建的场景：**
- 复杂业务逻辑（支付、库存、订单）
- 第三方 API 集成（需要隐藏密钥）
- 数据完全自主可控
- 用户量 > 10,000
- 需要定制化的搜索、推荐算法

### B. 常用命令速查

#### Flutter 命令
```bash
# 创建项目
flutter create my_app

# 运行项目
flutter run

# 热重载
r

# 清理构建
flutter clean

# 获取依赖
flutter pub get

# 生成代码（Freezed/JsonSerializable）
flutter pub run build_runner build --delete-conflicting-outputs

# 打包 APK
flutter build apk --release

# 打包 iOS
flutter build ios --release

# 分析代码
flutter analyze

# 运行测试
flutter test

# 查看设备
flutter devices
```

#### Git 命令
```bash
# 初始化仓库
git init

# 添加文件
git add .

# 提交
git commit -m "feat: add feature X"

# 推送
git push origin main

# 创建分支
git checkout -b feature/xxx

# 合并分支
git merge feature/xxx

# 查看状态
git status

# 查看日志
git log --oneline
```

#### Docker 命令（后端）
```bash
# 构建镜像
docker build -t my-api .

# 运行容器
docker run -p 8080:8080 my-api

# 查看容器
docker ps

# 停止容器
docker stop <container_id>

# 使用 docker-compose
docker-compose up -d

# 查看日志
docker-compose logs -f
```
