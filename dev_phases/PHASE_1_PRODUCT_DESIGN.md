## ✅ 任务清单

### Week 1-2：市场调研与需求分析（20-40小时）

#### 1. 市场调研（8-15小时）

- [ ] **确定产品方向**
  - [ ] 明确产品类型（工具类/社交类/内容类/电商类/其他）
  - [ ] 定义核心问题：这个产品解决什么问题？
  - [ ] 一句话描述产品价值主张

- [ ] **竞品分析**
  - [ ] 在 App Store 搜索相关关键词，找出 Top 10 竞品
  - [ ] 在 Google Play 搜索相关关键词，找出 Top 10 竞品
  - [ ] 深度体验 5 个主要竞品（下载并使用至少 30 分钟）
  - [ ] 记录竞品的优点和缺点
  - [ ] 分析用户评价中的高频痛点（至少 20 条评价）
  - [ ] 找出市场空白点和差异化机会

- [ ] **市场分析报告**
  - [ ] 使用 AI 生成市场分析报告
  - [ ] 评估市场规模和潜在用户数
  - [ ] 分析市场趋势和未来发展方向

**完成标准：**
- 竞品分析表格（至少 5 个竞品）
- 市场分析报告（1000-2000 字）
- 明确的产品差异化点（3-5 个）

---

#### 2. 产品定位（5-10小时）

- [ ] **目标用户画像**
  - [ ] 创建 3-5 个典型用户画像
  - [ ] 定义用户年龄、职业、使用场景
  - [ ] 分析用户痛点和需求
  - [ ] 描述用户的一天（Day in the Life）

- [ ] **价值主张**
  - [ ] 用一句话描述产品核心价值
  - [ ] 列出 3 个核心卖点
  - [ ] 说明与竞品的区别

- [ ] **产品定位声明**
  - [ ] 对于 [目标用户]
  - [ ] 遇到 [什么问题]
  - [ ] 我们的产品是 [产品类型]
  - [ ] 能够 [核心功能]
  - [ ] 不同于 [竞品]
  - [ ] 我们的产品 [差异化优势]

**完成标准：**
- 3-5 个详细的用户画像文档
- 清晰的产品定位声明
- 用户旅程地图（至少 1 个核心场景）

---

#### 3. 功能规划（7-15小时）

- [ ] **功能头脑风暴**
  - [ ] 列出所有可能的功能点（不限数量）
  - [ ] 与 AI 对话，扩展功能想法
  - [ ] 从用户视角审视每个功能的价值

- [ ] **功能分类（MoSCoW 方法）**
  - [ ] Must Have：MVP 必备功能（5-8 个）
  - [ ] Should Have：重要但可延后（3-5 个）
  - [ ] Could Have：锦上添花的功能（若干个）
  - [ ] Won't Have：本阶段不做的功能（记录备忘）

- [ ] **功能详细描述**
  - [ ] 为每个 Must Have 功能编写详细说明
  - [ ] 描述功能的输入、处理、输出
  - [ ] 列出功能的边界情况和异常处理
  - [ ] 评估技术实现难度（简单/中等/困难）

- [ ] **版本迭代规划**
  - [ ] v0.1（MVP）：包含哪些功能？
  - [ ] v1.0（正式版）：增加哪些功能？
  - [ ] v2.0（未来版）：长期规划

**完成标准：**
- 完整的功能清单（分类明确）
- MVP 功能详细说明文档
- 3 个版本的迭代路线图

### Week 3-4：UI/UX 设计（20-40小时）

#### 4. 信息架构设计（6-10小时）

- [ ] **页面结构规划**
  - [ ] 列出所有需要的页面（主页、列表页、详情页等）
  - [ ] 绘制页面层级关系图（用 Figma 或手绘）
  - [ ] 设计导航结构（Tab 导航/抽屉导航/堆栈导航）
  - [ ] 规划页面间的跳转关系

- [ ] **用户流程设计**
  - [ ] 绘制核心功能的用户操作流程图（至少 3 个）
  - [ ] 标注每个步骤的用户行为和系统反馈
  - [ ] 优化流程，减少不必要的步骤
  - [ ] 使用 AI 审查流程的合理性

- [ ] **信息层级规划**
  - [ ] 确定每个页面的信息优先级
  - [ ] 设计信息的展示方式（卡片/列表/网格等）

**完成标准：**
- 页面结构图
- 3 个核心用户流程图
- 导航结构设计方案

---

#### 5. 视觉设计（8-15小时）

- [ ] **设计风格探索**
  - [ ] 使用 Midjourney 生成 3-5 组设计灵感图
  - [ ] 确定设计风格（扁平/拟物/新拟态/极简等）
  - [ ] 收集灵感图片（Dribbble/Behance）

- [ ] **颜色系统**
  - [ ] 使用 AI 生成 3-5 组配色方案
  - [ ] 选择主色（Primary Color）
  - [ ] 选择辅助色（Secondary Color）
  - [ ] 定义中性色（灰度）
  - [ ] 定义语义色（成功/警告/错误/信息）
  - [ ] 验证颜色对比度（无障碍访问）

- [ ] **字体系统**
  - [ ] 选择标题字体（Font Family）
  - [ ] 选择正文字体
  - [ ] 定义字号层级（H1/H2/Body/Caption 等）
  - [ ] 定义行高和字重

- [ ] **间距系统**
  - [ ] 定义基础间距单位（如 4px/8px）
  - [ ] 创建间距比例（4/8/12/16/24/32/48/64）
  - [ ] 定义组件内边距和外边距规范

- [ ] **图标和图形**
  - [ ] 使用 AI 生成应用 Logo
  - [ ] 选择图标库（Material Icons/Font Awesome/自定义）
  - [ ] 设计或生成关键插图

- [ ] **设计规范文档**
  - [ ] 在 Figma 创建 Design System 页面
  - [ ] 整理颜色、字体、间距规范
  - [ ] 创建通用组件库（按钮/输入框/卡片等）

**完成标准：**
- 完整的颜色系统
- 字体和间距规范
- 应用 Logo（多种尺寸）
- Figma 设计规范文件

## 🤖 本阶段需要问 AI 的问题

### 市场调研阶段

**问题 1：市场分析**
```
我想开发一个 [应用类型]，目标用户是 [用户群体]。

请帮我：
1. 分析这个市场的规模和趋势（用数据说话）
2. 列出 5 个主要竞品及其优缺点（表格形式）
3. 找出市场空白点和机会
4. 提供 3 个差异化的产品方向建议
5. 评估每个方向的市场潜力和技术难度

输出格式：详细的市场分析报告（2000字左右）
```

**问题 2：用户画像生成**
```
我的产品定位是：[一句话描述]
核心功能：[列出 3-5 个功能]

请帮我创建 3 个典型用户画像，包括：
1. 基本信息（姓名/年龄/职业/收入）
2. 使用场景（什么时候/在哪里使用）
3. 痛点和需求（遇到什么问题）
4. 目标和动机（想达成什么）
5. 使用行为（技术水平/使用频率）
6. 一天的生活（描述一个典型的工作日）

输出格式：详细的用户画像卡片
```

**问题 3：功能规划**
```
我的产品定位是：[一句话描述]
目标用户：[用户画像]
核心价值：[解决什么问题]

请帮我：
1. 列出 MVP 必备的 8-10 个核心功能（Must Have）
2. 列出 5 个重要但可延后的功能（Should Have）
3. 列出若干个锦上添花的功能（Could Have）
4. 按开发优先级排序
5. 评估每个功能的开发难度（简单/中等/困难）
6. 估算每个功能的开发时间（小时）
7. 建议 3 个版本的功能迭代计划（v0.1/v1.0/v2.0）

输出格式：详细的功能清单（表格形式）+ 迭代路线图
```

**问题 5：配色方案生成**
```
我的应用是 [应用类型]，目标用户是 [用户群体]。

应用风格：[简洁现代/活泼有趣/专业严谨/...]

请帮我：
1. 生成 3-5 组配色方案
2. 每组包含：主色/辅助色/背景色/文字色/强调色
3. 提供十六进制色值

5. 验证颜色对比度（是否符合 WCAG 无障碍标准）
```

**问题 6：页面布局建议**
```

功能描述：[详细描述这个页面要展示什么、做什么]

需要的元素：
- [列出需要展示的内容和操作]

请帮我：
1. 建议页面布局方式（列表/网格/卡片/分栏等）
2. 描述信息层级和视觉重点
3. 建议交互方式（点击/滑动/长按等）
4. 提供参考案例（类似的优秀设计）
5. 给出 Flutter Widget 伪代码结构

输出格式：详细的布局方案 + Widget 代码结构
```

**问题 7：Midjourney 提示词生成**
```
我需要为 [应用类型] 应用设计 UI 灵感图。

风格要求：[简约/扁平/新拟态/渐变/...]
主色调：[颜色]
目标平台：iOS/Android

请帮我生成 5 个 Midjourney prompt，用于生成：
1. 应用整体界面设计
2. 主要页面布局
3. 配色方案参考
4. 图标和插图风格
5. 应用 Logo 设计

要求：
- 每个 prompt 包含风格、颜色、布局等关键词
- 适当使用参数（如 --ar 9:16 --v 6）
- 提供英文版本

输出格式：5 个完整的 Midjourney prompt
```

**问题 8：设计转代码**
```
我有以下设计稿：[描述设计或上传图片]

请帮我将设计转换为 Flutter Widget 代码框架：
1. 分析页面的组件结构（哪些是容器/列表/卡片等）
2. 提供完整的 Widget 树结构
3. 给出关键组件的代码实现
4. 标注颜色/字体/间距的具体数值
5. 建议响应式布局方案

要求：
- 使用 Material Design 3 组件
- 代码包含详细注释

输出格式：完整的 Flutter 代码 + 说明
```

## ⚠️ 常见问题和注意事项

### 避免的陷阱

1. **功能贪多**：不要试图在 MVP 中实现所有功能，专注核心价值
2. **过度设计**：设计要适度，不要在细节上纠结太久
3. **忽视用户**：不要闭门造车，尽早找真实用户验证想法
4. **技术驱动**：不要因为"技术酷"就加功能，要问"用户需要吗？"

### 决策检查清单

在进入下一阶段前，确保你能回答这些问题：

- [ ] 这个产品解决什么问题？（能用一句话说清楚）
- [ ] 目标用户是谁？（有具体的用户画像）
- [ ] 为什么用户会选择我而不是竞品？（差异化点明确）
- [ ] MVP 包含哪些功能？（5-8 个核心功能）
- [ ] 设计稿是否完整？（至少 5 个主要页面）
- [ ] 技术上可以实现吗？（没有明显的技术障碍）

### 如何验证想法

**方法 1：Landing Page 验证**
- 制作一个简单的产品介绍页
- 投放小额广告
- 看有多少人感兴趣（留邮箱/预注册）

**方法 2：用户访谈**
- 找 10 个目标用户
- 展示设计稿
- 询问他们是否会用、愿意付费多少
- 记录反馈并调整

**方法 3：MVP 原型测试**
- 使用 Figma 做一个可点击的原型
- 让用户尝试完成核心任务
- 观察卡点和困惑
- 迭代优化
