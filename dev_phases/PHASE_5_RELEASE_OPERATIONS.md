## ✅ 任务清单

### Week 29-30:发布准备(20-40小时)

#### 1. iOS App Store 发布(10-20小时)

- [ ] **开发者账号**(1-2小时)
  - [ ] 注册 Apple Developer 账号($99/年,约 ¥688)
  - [ ] 创建 App ID(Bundle Identifier)
  - [ ] 配置证书和描述文件(Xcode 自动管理)

- [ ] **应用信息**(3-6小时)
  - [ ] 应用名称(30 字符以内,使用 AI 优化)
  - [ ] 副标题(30 字符以内,突出卖点)
  - [ ] 应用描述(4000 字符,使用 AI 生成)
  - [ ] 关键词(100 字符,ASO 优化)
  - [ ] 应用分类(主分类和副分类)
  - [ ] 内容分级(年龄评级)
  - [ ] 隐私政策 URL(必须提供)
  - [ ] 支持 URL(可选)

- [ ] **应用素材**(4-8小时)
  - [ ] 应用图标(1024x1024 PNG,无圆角无透明)
  - [ ] 应用截图:
    - 6.5" iPhone(1284x2778,至少 3 张,最多 10 张)
    - 5.5" iPhone(1242x2208)
    - 12.9" iPad(可选)
  - [ ] 预览视频(可选,15-30 秒,竖屏)
  - [ ] 使用 AI 生成宣传图和文案

- [ ] **应用审核信息**(1-2小时)
  - [ ] 联系信息(姓名、电话、邮箱)
  - [ ] 演示账号(如需要登录)
  - [ ] 审核备注(特殊说明)

- [ ] **构建和提交**(2-4小时)
  - [ ] Xcode Archive 打包
  - [ ] 上传到 App Store Connect
  - [ ] 选择构建版本
  - [ ] 提交审核
  - [ ] 等待审核(通常 1-3 天)
  - [ ] 响应审核意见(如被拒)

**完成标准:**
- 应用已提交到 App Store
- 所有必填信息完整
- 等待审核结果

### 应用商店发布

**问题 1:应用商店描述文案**
```
我的应用信息如下:

应用名称:[名称]
应用类型:[工具类/社交类/...]
主要功能:[列出 5-8 个核心功能]
目标用户:[用户画像]
差异化优势:[与竞品相比的优势]

请为我生成:

1. **App Store / Google Play 应用描述**(吸引人的文案,800-1000 字)
   - 开头:吸引眼球的一句话
   - 核心功能介绍(带 emoji)
   - 用户价值说明
   - 使用场景举例
   - 结尾:行动号召

2. **简短描述**(80 字以内,精华浓缩)

3. **副标题/标语**(30 字以内,朗朗上口)

4. **关键词列表**(ASO 优化,20 个关键词)
   - 核心关键词
   - 长尾关键词
   - 相关关键词

5. **更新日志模板**(首个版本)

要求:
- 语言生动有趣
- 突出核心价值
- 包含社会证明(如适用)
- 结尾有明确的行动号召

输出格式:完整的应用商店文案包
```

## ⚠️ 常见问题和注意事项

### 审核常见被拒原因

**iOS App Store:**
1. 应用崩溃或有严重 bug
2. 功能不完整或与描述不符
3. 隐私政策缺失或不规范
4. 使用了私有 API
5. 界面设计不符合 HIG 指南
6. 元数据(截图、描述)与实际不符

**Android Google Play:**
1. 违反内容政策
2. 隐私政策缺失
3. 权限使用不当
4. 应用崩溃
5. 恶意行为

**快速通过审核的技巧:**
- 提供详细的审核说明
- 提供测试账号(如需登录)
- 确保应用无崩溃
- 隐私政策清晰可访问
- 及时回复审核团队问题
